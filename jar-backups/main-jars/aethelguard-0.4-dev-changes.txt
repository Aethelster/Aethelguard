Changes from 0.3.9-dev to 0.4-dev

Warning:
- This development build has not been tested in-game yet. Please test it on a backup/test server before using it on a live server.

- Created the 0.4-dev test build for the final free-release testing cycle.
- Hardened async authentication flows so Bukkit player actions and live auth state changes are safely returned to the main server thread.
- Added online-player guards to async login, register, PIN, and password-change callbacks so disconnecting during a check no longer continues stale success/error logic.
- Fixed PIN GUI login so existing correct PINs are checked against the stored hash instead of being rejected by the PIN policy validator.
- Locked the free-version PIN system to exactly 4 digits, matching the hardcoded keypad layout.
- Removed old configurable PIN length keys from the live config layout; old configs are cleaned by the config sync system.
- Fixed a rare 2FA edge case where a corrupted TOTP secret could fall back to an unsafe generated code.
- Added the required first-register security-question gate: when enabled, new players must answer one random recovery question before entering the server.
- Added a registration-complete popup and chat guidance explaining why the security question is needed.
- Added security-question bossbar/prompt support so the current required auth step stays visible.
- Allowed security-question answers through direct chat input as well as the command flow, while keeping the answer out of public chat and console blocked-chat logs.
- Fixed /login becoming ineffective after captcha when the live allowed-command config is missing or out of sync.
- Hardened password verification so empty, invalid, or corrupted stored password hashes produce a clear login error and console warning.
- Fixed the local YAML password storage schema: password hashes now live under password.hash, so password.usable no longer overwrites the actual hash.
- Added legacy local-password migration for older accounts that still stored the hash directly at password.
- Updated register, PIN register, player password change, admin password change, and recovery password updates to use the corrected local password hash path.
- Fixed config layout sync for empty lists so keys like manual-suspicious-ips and custom cooldown commands stay as [] instead of being treated as missing on every reload.
- Fixed auth-settings.reload.reauth-online-players so /aethelguard reload can send online players back to the auth screen when enabled.
- Fixed stale auth-timeout tasks so older timeout checks cannot log or kick a player before the current timeout deadline expires.
- Changed the default normal auth timeout from 60 seconds to 90 seconds.
- Added separate 2FA timeout handling: 2FA accounts get a hardcoded 30-second captcha-stage timeout, then the timeout resets to auth-settings.two-factor.timeout-ticks once the 2FA code step starts.
- Added a hardcoded 30-second security-question timeout after 2FA when an account still needs to finish the required registration question.
- Documented the default 2FA recovery flow in config.yml as 30s captcha + 60s 2FA code + 30s security question.
- Added auth-settings.two-factor.timeout-ticks to config.yml and config sync defaults; the temporary configurable 2FA captcha-timeout key is removed automatically because the 30-second captcha stage is fixed.
- Fixed half-finished registration recovery: if a player registered but timed out before saving the required security question, the question is shown again after their next successful login/PIN/2FA instead of leaving the account incomplete.
- Prevented duplicate first captcha and registration security-question prompt messages by delaying the first repeat for states that already send an entry message.
- Added normal chat input support for captcha answers while still blocking the message from public chat.
- Locked MAP captcha players to hotbar slot 1, forced the captcha view downward, blocked offhand swapping, item dropping, and inventory movement while unauthenticated.
- Changed MAP captcha delivery to place the map directly in slot 1 and remove it cleanly after captcha/auth state ends.
- Disabled player security-question changes once a question is already saved, while keeping the first-register question flow active.
- Added a PIN GUI-specific bossbar message and stopped regular PIN spam while the GUI keypad is open.
- Added five-close protection for the PIN GUI: closing the keypad repeatedly before auth now kicks the player after 5 closes.
- Fixed the PIN GUI timeout counter so it no longer appears to close early while showing 2 seconds remaining.
- Fixed the PIN GUI bossbar for both PIN login and first-time PIN setup GUI states.
- Changed the default PIN GUI success sound to the classic player level-up sound.
- Added a dedicated two-factor login success sound.
- Added a QR-code setup link to the 2FA setup message while keeping the raw secret visible as a backup.
- Made the 2FA QR setup message use a real MiniMessage click event so players can click the QR label in chat.
- Fixed the 2FA QR setup link so the URL no longer gets corrupted by legacy color-code conversion and no longer crashes system chat packet encoding.
- Added a general URL guard to message formatting so legacy color-code conversion no longer rewrites ampersands inside http/https links.
- Simplified the 2FA setup message so the QR URL is clickable through a short "Open with QR" / "QR ile ac" label instead of printing the long link in chat.
- Reworked raw/kick-screen message formatting so MiniMessage tags such as <gray> are parsed before being shown on disconnect screens.
- Replaced hardcoded timeout text in messages with the {seconds} placeholder where needed.
- Updated bossbar text to show the current auth step and remaining seconds for captcha/login/register/PIN/setpin/2FA/security-question states, while leaving the PIN GUI bossbar unchanged.
- Added adaptive security chat notices for trusted-IP captcha bypass and suspicious-IP extra captcha.
- Cleared player chat when the first-register security-question step begins so the required recovery question is easier to notice.
- v0.4-dev is still a pre-release and should not be treated as the final stable version.
- v0.4-sentinel will be the final free release after in-game testing and feedback.
