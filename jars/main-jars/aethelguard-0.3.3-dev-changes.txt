Changes from 0.3.2-dev to 0.3.3-dev

- Added PIN GUI polish for the free 0.4 development line.
- Added theme secret heads for hidden PIN input display.
- Added auth-settings.pin.gui.hide-input so admins can choose hidden or visible PIN digits.
- Added auth-settings.pin.gui.randomize-numbers so admins can optionally shuffle keypad numbers per GUI open.
- Changed default PIN settings so PIN auth and PIN GUI are disabled by default, keeping password login/register as the default flow.
- Changed the default PIN GUI theme to quartz.
- Removed the configurable PIN GUI filler material and hardcoded the keypad filler to a stable black stained glass pane.
- Added passive confirm behavior: the confirm button stays disabled until the configured minimum PIN length is reached.
- Added a short wrong-PIN GUI error state and submit lock to prevent repeated confirm spam.
- Added dedicated PIN GUI sounds:
  - open
  - close
  - wrong PIN
  - disabled confirm
  - GUI success with villager trade sound
- Kept normal /pin and /setpin command sounds separate from GUI success sounds.
- Added /aethelguard pingui preview <theme> for admins to preview PIN GUI themes without enabling PIN login.
- Added tab completion for the PIN GUI preview command and theme names.
- Added missing English and Turkish message keys for PIN GUI preview, visible digits, wrong input, and disabled confirm states.
- Updated the automatic config default system so the new PIN GUI settings and sounds are added to existing configs.
- Updated wrong PIN GUI feedback so the keypad filler/background turns red for 5 seconds after an incorrect GUI PIN attempt.
